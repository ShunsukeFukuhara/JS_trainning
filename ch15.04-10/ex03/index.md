### 問題

ブラウザの開発者ツールを使うと CSS のデバッグを効率的に行うことができる。
CSS のデバッグ を参考にして以下を実施しなさい:

1. 15.4-10.1 および 15.4-10.2 の ToDo アプリに対してブラウザの開発者ツールから値の変更やプロパティの追加を試してみなさい
2. 開発者ツールで CSS に関して実行できる操作を検索エンジンで調べ、便利だと思ったものを 3 つ挙げなさい
3. 15.4-10.2 のアプリの body 要素に対し、元々 HTML および JS 内で利用していなかった Tailwind CSS のクラス (bg-rose-600 など何でも良い) を開発者ツールから追加すると変更が反映されないが、これは何故か調べなさい

### 解答

1. ブラウザで実行。15.4-10.1でタイトルと枠が被った部分があり、formにmargin-topを追加して修正。15.4-10.2ではToDoアイテムの背景色を変更。15.4-10.2では、TODOアイテムの背景色を変えるなどした。
2. [https://ics.media/entry/230317/](https://ics.media/entry/230317/)を参照し、以下の3つを挙げる。
   - :hover時のスタイルを確認する
     - この機能を使うと、要素にマウスオーバーしたときのスタイルを簡単に確認・編集できる。ホバー時に毎回マウスを動かして確認していた手間が省ける。
   - 影を調整する
     - 要素の影をリアルタイムで調整できる。影の大きさ、ぼかし、色などを視覚的に変更できるため、デザインの微調整が容易になる。
   - ライトモード / ダークモードをシミュレーションする
     - ブラウザがダークモードでどのように表示されるかを確認できる。毎回ブラウザの設定を切り替える手間が省ける。
3. Tailwind CSSでは、使用しているクラスに対応するスタイルが事前に生成される仕組みがある。開発者ツールで後からクラスを追加しても、そのクラスに対応するスタイルが生成されていないため変更が反映されない。このような仕組みは「「JIT (Just-In-Time) モード」と呼ばれ、使用されていないクラスを削除してファイルサイズを小さくするために利用される。
